## Role detection & sub-datapack overlays
* Role is auto-detected from the ROM header (`GameInfo::detect`), NOT fuzzy similarity. Canonical base = (code, version, 16MiB) + valid `gMapGroups`. Off-fingerprint (32MiB / relocated tables) = standalone hack main, label = filename slug; its `gMapGroups` is signature-scanned (`Decoder::findMapGroups`). Per-game enrichment tables (wild/trainers/names/doors/OW gfx) are OPTIONAL ‚Äî skip gracefully (no garbage) when 0.
* Sub = an official sibling sharing a region (geometry-identical): Kanto main `firered` + sub `leafgreen`; Hoenn main `ruby` + subs `sapphire`, `emerald`. A sub goes to `map/main/<main>/sub/<sub>/`. Hacks (glazed/hns) are standalone mains, never subs.
* A sub overlay is DIFF-ONLY: no `.tmx`/tileset (geometry shared from main); write `informations.xml` + a PARTIAL per-map `<map>.xml` holding ONLY the changed wild sections (`<grass>`/`<water>`), diffed against the already-generated main. Matches `datapack-pkmn/map/main/test/sub/smallchange/`. So the main must be generated before its subs.

## Tileset model (TilesetBuilder)
* ALWAYS think about how a HUMAN sees it: a human sees a single OBJECT (a building, a tree), but in the ROM that object is often tiles on 2 DIFFERENT layers (the ground/under tile + the over/WalkBehind top tile). Group and lay out tiles by the MERGED human-visible object, not per-layer ‚Äî a building's ground tiles AND its over tiles belong together in the sheet (the over's roof sits above its ground's wall), as seen.
* Pack tiles as rigid 2-D blocks by their DOMINANT (most-frequent, reciprocated) on-map neighbour so a building reads like the map. Place blocks with a SKYLINE bottom-left strip bin-packer, then FILL every gap with the free single tiles. Sheet WIDTH is ADAPTIVE per pool (`‚âàceil(sqrt(tiles)*1.15)`, clamped `[8, kColumns=32]`) so small pools don't waste a 32-wide sheet (cell ids stay linear ‚Üí gid math unchanged). Target: low transparent-space (firered ~20%).
* Fold NEAR-duplicate tiles (`dedupAdd`): the ROM stores metatiles that render visually-identical ‚Äî fold when every RGBA channel within `kNearMaxDiff` and mean within `kNearMeanDiff`. Animation frames never fold.
* Compose vs decompose ACROSS the two metatile sub-layers (refines the root's "collidable over ‚Üí 2nd Collisions" rule): a UNIQUE top on an always-collidable cell (a building face) is COMPOSITED into one ground tile (`renderMetatile`, no over). A REUSABLE overlay (top sits on ‚â•2 backgrounds OR its base terrain also exists standalone ‚Äî cliff/rock/tree/sign) stays a SEPARATE transparent over tile over the base terrain (reused over any background). Walkable tops still lift to WalkBehind (player passes behind).
* Background/foreground SPLIT of BAKED tiles (Pass 1b, image-based not sub-layer): the ROM sometimes bakes a collidable object onto a terrain in ONE image, so the same object is several near-dups differing only in background. For a composited, always-collidable, fully-opaque, normal-behaviour ground tile, find a terrain T that is WALKABLE somewhere (so the split is not inverted) and matches it except a connected foreground ‚â•12 px ‚Üí split into under=T (reused) + over=the transparent object. Keep only foregrounds reused over ‚â•2 DISTINCT terrains. Collidable ‚áí T on Collisions, object on 2nd Collisions (below player, OR-merged) ‚Äî background never lands on the Water/Grass logic layer. `kBgMatchDiff`/`kMinForeground`/`kMinBackground`; helpers `foregroundOver`/`isFullyOpaque`.
* Region subfolders `tileset/<region>/` only when a label has >50 pools AND spans ‚â•2 regions (region-specific pools nest, shared pools stay at `tileset/` root); single-region labels stay flat. `TilePool.subDir`; `tilesetRefs` prefixes it (CCWriter unchanged).
* Door + animated tiles go on their OWN sheet per pool, named `<base>_anim_<n>` (separate from the ground/over `<base>_<n>` sheets) so they are grouped on a unique, easy-to-find tileset. Implementation: tiles stay laid out grounds/overs (`mainCount`) then door+anim; `poolSheetSegments(mainCount,uniqueCount)` cuts the main sheets at `kCapacity` then the anim sheet(s); gids stay `base+position`, firstgid per sheet is `base+startCell` (so `secondaryBase`/`markerGid` use `uniqueCount`, not `sheetCount*kCapacity`).
* Each `.tsx` carries a best-effort Tiled `<wangset type="corner">` (`emitWangSet`): opaque tiles' 8√ó8 corners clustered by average colour ‚Üí dominant terrains ‚Üí per-tile `wangid`. Editor-only (libtiled ignores it when rendering); knobs `kJoin`/`minCount`/cap.
