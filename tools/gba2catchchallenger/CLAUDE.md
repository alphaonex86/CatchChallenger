# gba2catchchallenger — rules

Combine with the root CLAUDE.md (esp. "a ROM converter writes ONLY inside `map/main/<label>/`", disjoint real-tile feature layers, relative `.tsx` paths). Build dir out of tree (`/tmp/gba2cc-build`). Every change must keep ALL guards PASS: TilesetBuilder `dup` + `adjacency` + `layer-recompose` (EVERY tile split across 2 layers — terrain+object overlay, Pass-1b bg/fg split, building wall+roof — must recompose under+over back to the ROM metatile; bg/fg objects ≥12 px), CCWriter `layer-visibility` + `top-layer-cover` + `render-visibility` (libtiled re-renders every `.tmx`).
WORKFLOW after ANY code change: compile → run the converter on one ROM and confirm EVERY guard PASSes, FIX until it does → ONLY THEN regenerate the full datapack/maps. Never regenerate maps on a build whose guards still FAIL. The two 32MB hacks (glazed/hns) keep a tiny pre-existing render-visibility miss (best-effort on generic-named maps); the aggressive bg/fg pair-split (Pass 1c) raises hns's miss a little (~2→7 / 4179 layers) — accepted on the hack, officials (firered/ruby) stay fully clean.

## Role detection & sub-datapack overlays
* Role is auto-detected from the ROM header (`GameInfo::detect`), NOT fuzzy similarity. Canonical base = (code, version, 16MiB) + valid `gMapGroups`. Off-fingerprint (32MiB / relocated tables) = standalone hack main, label = filename slug; its `gMapGroups` is signature-scanned (`Decoder::findMapGroups`). Per-game enrichment tables (wild/trainers/names/doors/OW gfx) are OPTIONAL — skip gracefully (no garbage) when 0.
* Sub = an official sibling sharing a region (geometry-identical): Kanto main `firered` + sub `leafgreen`; Hoenn main `ruby` + subs `sapphire`, `emerald`. A sub goes to `map/main/<main>/sub/<sub>/`. Hacks (glazed/hns) are standalone mains, never subs.
* A sub overlay is DIFF-ONLY: no `.tmx`/tileset (geometry shared from main); write `informations.xml` + a PARTIAL per-map `<map>.xml` holding ONLY the changed wild sections (`<grass>`/`<water>`), diffed against the already-generated main. Matches `datapack-pkmn/map/main/test/sub/smallchange/`. So the main must be generated before its subs.

## Tileset model (TilesetBuilder)
* ALWAYS think about how a HUMAN sees it: a human sees a single OBJECT (a building, a tree), but in the ROM that object is often tiles on 2 DIFFERENT layers (the ground/under tile + the over/WalkBehind top tile). Group and lay out tiles by the MERGED human-visible object, not per-layer — a building's ground tiles AND its over tiles belong together in the sheet (the over's roof sits above its ground's wall), as seen.
* Pack tiles as rigid 2-D blocks by their DOMINANT (most-frequent, reciprocated) on-map neighbour so a building reads like the map. Place blocks with a SKYLINE bottom-left strip bin-packer, then FILL every gap with the free single tiles. Sheet WIDTH is ADAPTIVE per pool (`≈ceil(sqrt(tiles)*1.15)`, clamped `[8, kColumns=32]`) so small pools don't waste a 32-wide sheet (cell ids stay linear → gid math unchanged). Target: low transparent-space (firered ~20%).
* Fold NEAR-duplicate tiles (`dedupAdd`): the ROM stores metatiles that render visually-identical — fold when every RGBA channel within `kNearMaxDiff` and mean within `kNearMeanDiff`. Animation frames never fold.
* Compose vs decompose ACROSS the two metatile sub-layers (refines the root's "collidable over → 2nd Collisions" rule): a UNIQUE top on an always-collidable cell (a building face) is COMPOSITED into one ground tile (`renderMetatile`, no over). A REUSABLE overlay (top sits on ≥2 backgrounds OR its base terrain also exists standalone — cliff/rock/tree/sign) stays a SEPARATE transparent over tile over the base terrain (reused over any background). Walkable tops still lift to WalkBehind (player passes behind).
* Background/foreground SPLIT of BAKED tiles (Pass 1b, image-based not sub-layer): the ROM sometimes bakes a collidable object onto a terrain in ONE image, so the same object is several near-dups differing only in background. For a composited, always-collidable, fully-opaque, normal-behaviour ground tile, find a terrain T that is WALKABLE somewhere (so the split is not inverted) and matches it except a connected foreground ≥12 px → split into under=T (reused) + over=the transparent object. Keep only foregrounds reused over ≥2 DISTINCT terrains. Collidable ⇒ T on Collisions, object on 2nd Collisions (below player, OR-merged) — background never lands on the Water/Grass logic layer. `kBgMatchDiff`/`kMinForeground`/`kMinBackground`; helpers `foregroundOver`/`isFullyOpaque`.
* PAIR-BASED partial match (Pass 1c) catches a SAME object on DIFFERENT backgrounds that the full-tile terrain match (1b) misses — its min-foreground pick is often NOT the real background (e.g. common_0 #315 cliff-over-water + #319 cliff-over-sand). The SHARED connected region of two baked tiles (`sharedObjectMask`) is their common OBJECT, consistent for both (a per-tile min-fg mask is not); the REST is background, PARTIAL-matched over the object's complement against WALKABLE terrains ONLY — the discriminator that keeps the object the foreground (if the shared region were itself background, the "background" would be the real object, which matches no walkable terrain → rejected). Pre-check the recomposition (terrain+object == ROM tile) before applying, require the object reused over ≥2 DISTINCT backgrounds, `kMinObject=40`. Goal: group the OBJECT, drop the background to the lower (Collisions) layer.
* When tiles can NOT be grouped by map adjacency (leftover singles), group them by another findable characteristic — `tileCategory`: feature (water/grass/ledge by behaviour) / building (collidable) / background (walkable, the common case) / overlay — so the un-anchored singles cluster by kind in the sheet instead of scattering. Passed into `layout2D` as `catOf`; the smart gap-fill anchors to map-neighbours first, then sorts the rest by category.
* Region subfolders `tileset/<region>/` only when a label has >50 pools AND spans ≥2 regions (region-specific pools nest, shared pools stay at `tileset/` root); single-region labels stay flat. `TilePool.subDir`; `tilesetRefs` prefixes it (CCWriter unchanged).
* Door + animated tiles go on their OWN sheet per pool, named `<base>_anim_<n>` (separate from the ground/over `<base>_<n>` sheets) so they are grouped on a unique, easy-to-find tileset. Implementation: tiles stay laid out grounds/overs (`mainCount`) then door+anim; `poolSheetSegments(mainCount,uniqueCount)` cuts the main sheets at `kCapacity` then the anim sheet(s); gids stay `base+position`, firstgid per sheet is `base+startCell` (so `secondaryBase`/`markerGid` use `uniqueCount`, not `sheetCount*kCapacity`).
* Each `.tsx` carries a best-effort Tiled `<wangset type="corner">` (`emitWangSet`): opaque tiles' 8×8 corners clustered by average colour → dominant terrains → per-tile `wangid`. Editor-only (libtiled ignores it when rendering); knobs `kJoin`/`minCount`/cap.
