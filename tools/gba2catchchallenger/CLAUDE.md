# gba2catchchallenger — rules

Combine with the root CLAUDE.md (esp. "a ROM converter writes ONLY inside `map/main/<label>/`", disjoint real-tile feature layers, relative `.tsx` paths). Build dir out of tree (`/tmp/gba2cc-build`). Every change must keep ALL guards PASS on the OFFICIAL ROMs: TilesetBuilder `dup` + `adjacency` + `layer-recompose` (EVERY tile split across 2 layers — terrain+object overlay, Pass-1b bg/fg split, building wall+roof — must recompose under+over back to the ROM metatile; bg/fg objects ≥12 px) + `tiny-tile` + `metatile-routing` (every map cell's metatile must decode to graphics in the array it is routed to — primary if id<`metatilesInPrimary` else secondary; FAIL = the routed array is empty but the OTHER holds graphics at that id ⇒ wrong `metatilesInPrimary` split ⇒ "black hole" cell), CCWriter `layer-visibility` + `top-layer-cover` + `render-visibility` (libtiled re-renders every `.tmx`).
WORKFLOW after ANY code change: compile → run the converter on one ROM and confirm EVERY guard PASSes, FIX until it does → ONLY THEN regenerate the full datapack/maps. Never regenerate maps on a build whose guards still FAIL. `metatile-routing` is 0 (PASS) on EVERY ROM now (officials + both hacks): an expanded-tileset hack that kept one engine's MAP format but the other's bigger TILESETS (HnS = Emerald maps + FRLG-size 640 tilesets) is auto-corrected — `autodetectTilesetSplit` (main.cpp) decodes every cell under each standard split (512/512/6 vs 640/640/7) and picks the one with fewest empty metatiles (HnS 512→640, officials/Glazed unchanged), so high-id building metatiles route to their real primary graphics instead of empty secondary slots. The guard's `primary[id]` probe is bounded by the primary's DEFINED count `(attrPtr-metaPtr)/16` so an in-bounds-only check has no OOB false positives. This also CLEARED HnS's old render-visibility miss — HnS now passes ALL guards. Glazed keeps a tiny pre-existing layer-/render-visibility miss on its generic-named maps (best-effort, NOT a metatile-split issue; auto-detect leaves Glazed at 512 — its output is unchanged). The aggressive bg/fg PAIR-split (Pass 1c) is SKIPPED for the hacks (`rom_.size()>16MB`) because their generic tiles over-extract and worsen that miss — officials (≤16MB) get the full pair-split.

## Role detection & sub-datapack overlays
* Role is auto-detected from the ROM header (`GameInfo::detect`), NOT fuzzy similarity. Canonical base = (code, version, 16MiB) + valid `gMapGroups`. Off-fingerprint (32MiB / relocated tables) = standalone hack main, label = filename slug; its `gMapGroups` is signature-scanned (`Decoder::findMapGroups`). Per-game enrichment tables (wild/trainers/names/doors/OW gfx) are OPTIONAL — skip gracefully (no garbage) when 0.
* The engine default tileset split (RSE 512/512/6, FRLG 640/640/7) is WRONG for a hack that kept one engine's MAP format but expanded its TILESETS to the other's size (HnS = Emerald maps + FRLG 640 tilesets → black-hole cells). `autodetectTilesetSplit` (main.cpp, run after `decodeAll`, before tilesets) decodes every map cell under each standard split and overrides `metatilesInPrimary`/`tilesInPrimary`/`palettesInPrimary` only on a STRICT empty-cell reduction — retail ROMs are never touched, HnS flips 512→640. Each tileset's DEFINED primary metatile count is `(attrPtr-metaPtr)/16` (attr array follows the metatile array).
* NAMES for a hack (`detect()` left `mapNameTable==0`): `recoverMapNameTable` (main.cpp, after `decodeAll`, before Naming) finds the RELOCATED section-name table by scanning for the stride-8 base (name ptr at +4) whose entries best TYPE-MATCH the maps (a town map names a `…TOWN/CITY`, a route `ROUTE …`, a cave `CAVE/TUNNEL/WELL/…`) — a wrong base or a generic string pool scores ~0; needs score≥8. Recovers HnS=Johto@0x96db40, Glazed@0x5a147c → real area names AND grouping (the 844-map `area-0` blob splits by `regionSection`). `Gen3Text::strictName` validates an entry (no dropped control bytes). Then `namedSidOf` falls back to a map's OWN named section when no warp-reachable named area exists (a detached building keeps its town). FINAL fallback for a truly-unnamed group (`Naming.cpp`): the Gen3 `mapType` decides `city-N`/`cave-N`/`road-N` — NOT `area-N`; every isolated unnamed INTERIOR (no name, no warp to any area) merges under ONE shared `building/` parent (`building-N`/`house-N` inside), never a top-level sibling of the real towns. (Similar-map / sign-text detection is unneeded: the recovered table gives EXACT names.)
* REGION top-folder (`<label>/<region>/<area>/`, `Naming.cpp`, hacks only — retail keeps `regionFor` kanto/sevii/hoenn): a multi-region hack (HnS=Johto+Kanto) splits by (1) `regionFromName` keyword table (town/landmark + route-number kanto 1-28 / johto 29-48), (2) PROPAGATE along the warp/connection graph (an unnamed cave/road inherits the region of the town it links to), (3) a leftover isolated cluster ≥30 maps = an undetected region → `region-1`/`region-2` (the shared `building/` bucket folds into the dominant region, never its own). Cross-region warps resolve via `relPath` (same `../../<region>/…` form retail already uses for kanto↔sevii). Hack = `gi.region==gi.label`. Datapack format already supports `<region>/` + `region-N` buckets (the engine keys on the PATH) — no format change; original datapack tree is read-only.
* Sub = an official sibling sharing a region (geometry-identical): Kanto main `firered` + sub `leafgreen`; Hoenn main `ruby` + subs `sapphire`, `emerald`. A sub goes to `map/main/<main>/sub/<sub>/`. Hacks (glazed/hns) are standalone mains, never subs.
* A sub overlay is DIFF-ONLY: no `.tmx`/tileset (geometry shared from main); write `informations.xml` + a PARTIAL per-map `<map>.xml` holding ONLY the changed wild sections (`<grass>`/`<water>`), diffed against the already-generated main. Matches `datapack-pkmn/map/main/test/sub/smallchange/`. So the main must be generated before its subs.

## Engine-fit normalisation (Decoder::finalizeMaps — after decodeAll, before everything)
* Warp dests resolved to absolute (destX,destY) (dest warp-ids break after a split); CCWriter clamps dest into the dest grid and SKIPS warps whose SOURCE is off-map (RSE ferry pads at x==width).
* ONE connection per map side (engine border limit): keep the largest shared edge, then drop non-reciprocal leftovers — verdicts computed against the ORIGINAL lists BEFORE filtering (filtering while checking corrupts the stored indices).
* Maps >127 tiles (COORD_TYPE limit) split into chunks along the long axis: plan ids → retarget every warp/conn (so chunk0's reused id is unambiguous) → build chunks (blocksOverride in-memory grid, read cells ONLY via DecodedMap::blockAt; synthetic inter-chunk borders use final ids).  origMap keeps the ROM id for wild lookups.
* Wild land weights are 20,20,10,10,10,10,5,5,4,4,1,1 (=100); emitWildList rescales to EXACTLY 100 (engine drops the whole list otherwise).  Border markers clamp into [max(0,O), min(thisDim-1,neighDim-1+O)] — symmetric, keeps the pair difference = the Gen3 offset and the object inside the map.

## Tileset model (TilesetBuilder)
* ALWAYS think about how a HUMAN sees it: a human sees a single OBJECT (a building, a tree), but in the ROM that object is often tiles on 2 DIFFERENT layers (the ground/under tile + the over/WalkBehind top tile). Group and lay out tiles by the MERGED human-visible object, not per-layer — a building's ground tiles AND its over tiles belong together in the sheet (the over's roof sits above its ground's wall), as seen.
* Pack tiles as rigid 2-D blocks by their DOMINANT (most-frequent, reciprocated) on-map neighbour so a building reads like the map. Place blocks with a SKYLINE bottom-left strip bin-packer, then FILL every gap with the free single tiles. Sheet WIDTH is ADAPTIVE per pool (`≈ceil(sqrt(tiles)*1.15)`, clamped `[8, kColumns=32]`) so small pools don't waste a 32-wide sheet (cell ids stay linear → gid math unchanged). Target: low transparent-space (firered ~20%).
* Fold NEAR-duplicate tiles (`dedupAdd`): the ROM stores metatiles that render visually-identical — fold when every RGBA channel within `kNearMaxDiff` and mean within `kNearMeanDiff`. Animation frames never fold.
* Compose vs decompose ACROSS the two metatile sub-layers (refines the root's "collidable over → 2nd Collisions" rule): a UNIQUE top on an always-collidable cell (a building face) is COMPOSITED into one ground tile (`renderMetatile`, no over). A REUSABLE overlay (top sits on ≥2 backgrounds OR its base terrain also exists standalone — cliff/rock/tree/sign) stays a SEPARATE transparent over tile over the base terrain (reused over any background). Walkable tops still lift to WalkBehind (player passes behind).
* Background/foreground SPLIT of BAKED tiles (Pass 1b, image-based not sub-layer): the ROM sometimes bakes a collidable object onto a terrain in ONE image, so the same object is several near-dups differing only in background. For a composited, always-collidable, fully-opaque, normal-behaviour ground tile, find a terrain T that is WALKABLE somewhere (so the split is not inverted) and matches it except a connected foreground ≥12 px → split into under=T (reused) + over=the transparent object. Keep only foregrounds reused over ≥2 DISTINCT terrains. Collidable ⇒ T on Collisions, object on 2nd Collisions (below player, OR-merged) — background never lands on the Water/Grass logic layer. `kBgMatchDiff`/`kMinForeground`/`kMinBackground`; helpers `foregroundOver`/`isFullyOpaque`.
* PAIR-BASED partial match (Pass 1c) catches a SAME object on DIFFERENT backgrounds that the full-tile terrain match (1b) misses — its min-foreground pick is often NOT the real background (e.g. common_0 #315 cliff-over-water + #319 cliff-over-sand). The SHARED connected region of two baked tiles (`sharedObjectMask`) is their common OBJECT, consistent for both (a per-tile min-fg mask is not); the REST is background, PARTIAL-matched over the object's complement against WALKABLE terrains ONLY — the discriminator that keeps the object the foreground (if the shared region were itself background, the "background" would be the real object, which matches no walkable terrain → rejected). Pre-check the recomposition (terrain+object == ROM tile) before applying, require the object reused over ≥2 DISTINCT backgrounds, `kMinObject=64`, `kMinBackground=36` (a big object may leave only a small border background — common_0 #360/#365 boulder, bg 38px). SKIPPED for >16MB hacks. Goal: group the OBJECT, drop the background to the lower (Collisions) layer.
* When tiles can NOT be grouped by map adjacency (leftover singles), group them by another findable characteristic — `tileCategory`: feature (water/grass/ledge by behaviour) / building (collidable) / background (walkable, the common case) / overlay — so the un-anchored singles cluster by kind in the sheet instead of scattering. Passed into `layout2D` as `catOf`; the smart gap-fill anchors to map-neighbours first, then sorts the rest by category.
* Region subfolders `tileset/<region>/` only when a label has >50 pools AND spans ≥2 regions (region-specific pools nest, shared pools stay at `tileset/` root); single-region labels stay flat. `TilePool.subDir`; `tilesetRefs` prefixes it (CCWriter unchanged).
* A shared (`common`) tileset gets a readable CATEGORY suffix from its maps' ROM `mapType` — `common-outdoor` / `common-indoor` / `common-cave` instead of `common-7`. `poolCategory()`: OUTDOOR is the overworld BASE (a general tileset spans outdoor+cave+indoor maps, e.g. firered general = 76 outdoor/80 cave/35 indoor), so outdoor when its outdoor use is ≥30%; else cave/indoor by majority. Numeric suffix still resolves collisions.
* Ambient animations (water/flowers/lava — `registerGlobalAnim(...,ambient=true)`) ALSO carry a standard Tiled `<animation>` (frames follow the base tile in the row) so the editor previews the loop; DOORS do NOT (they animate on player action only).  ALL water/door animations go to ONE shared GLOBAL anim tileset per label `tileset/anim_<n>`, de-duplicated ACROSS pools (water/flowers/doors appear once), laid out 1 ANIMATION PER ROW (`globalAnimCols_` = max(waterFrames,8) wide). A pool's own tiles are just grounds+overs (`mainCount==uniqueCount`, no per-pool anim region); it records only `animGlobal[metatile]`→global index. gids: `groundGid`(water)/`doorGid` return `animBase(map)+globalIndex`; `animBase` = after primary+secondary pools (`secondaryBase + secondary.uniqueCount`); `markerGid` = `animBase + globalAnims_.size()`; `tilesetRefs` adds `anim_<n>` at `animBase`. Built via `registerGlobalAnim` (dedup by frame content) during buildPool, emitted once by `emitGlobalAnim` in prepare. (firered: 62 animations in one anim_0 vs ~22 tiny per-pool sheets before.)
* Each `.tsx` carries a best-effort Tiled `<wangset type="corner">` (`emitWangSet`): opaque tiles' 8×8 corners clustered by average colour → dominant terrains → per-tile `wangid`. Editor-only (libtiled ignores it when rendering); knobs `kJoin`/`minCount`/cap.

## --all : whole self-contained datapack (Gen3Data + FullWriter + SpriteRipper)
Default mode writes ONLY maps (overlay onto a base datapack) — but the maps reference Pokemon/items BY NAME, so the base must define them; without it the encounters/trainers DANGLE. `--all <root>` additionally extracts the WHOLE game DB to the datapack ROOT so it is self-contained.
* **Auto-locate, never hard-code offsets.** `Gen3Data::decode` finds each table by anchoring on known bytes, so it works on every retail ROM (FR/LG/Ru/Sa/Em — all different offsets): gBaseStats via Bulbasaur `45 49 49 45 65 65 0C 03 2D 40` (−28); gBattleMoves via Pound `00 28 00 64 23` + the power sequence 40/50/15/18/80 (−12); gMoveNames via encoded "POUND"+FF (−13); gEvolutionTable via `04 00 10 00 02 00` (−40, NOT −30: each Evolution is **8 bytes** {method,param,species,pad}, 5 slots = 40/species); gTypeEffectiveness via `00 05 05 00 08 05`; gLevelUpLearnsets by the pointer-array-of-learnsets scan (entry u16 = level<<9 | move); gItems via encoded "MASTER BALL"+FF (−44) — DON'T trust `GameInfo.itemNames`, it is only set for FR+Ru, the other 3 read garbage. Skip the 25 unused internal species slots (252-276) by `hasAlpha(name)`. Items: iterate ≤377, skip non-alpha/price>60000 (FRLG has a big mid-table unused gap 226-253, so a blank-run terminator stops too early).
* **Stat scale:** cc = 2*base+110 (HP) / 2*base+5 (others), ≈ the real lvl-100 stat. catch=base/2.55, eggStep=eggCycles*256, ratio_gender=genderRatio/256*100 (255→-1), give_xp=baseExp*40. Evolution method 4/8-15→level(param), 7→item(param), 1-3/5-6 (friendship/trade)→level 36.
* **SpriteRipper:** gMonFrontPicTable/gMonBackPicTable (tag==index + ptr→0x10 LZ77, 8-byte recs {ptr,u16 size,u16 tag}), gMonPaletteTable ({ptr,u16 tag,u16}). LZ77 → 64×64 4bpp GBA tiles (8×8 tile grid, byte=2px low-nibble-first) + 16-colour BGR555 palette (colour 0 = transparent) → front/back/small.png. **`GbaRom::pointer(off)` reads the pointer AT file offset `off`** — pass the field offset, NOT the already-read u32 value.
* FullWriter also writes the completeness plumbing (start.xml with starter species + Poke Ball id 4 + a placeholder player skin, reputation/event/layers/visualcategory/fight-bg, empty plants/crafting/buff) so the datapack is startable. Verify: parseDatapack → profiles=1 skins=1, 0 dangling map refs, maps load.
* **Music (M4aRipper):** a from-scratch MP2k/"Sappy" software player. Locate gSongTable via the `SelectSong` Thumb signature (SIG_A/SIG_B) then read the table ptr at +40 (NOT a content scan — that hits decoys). Interpret each track: tempo accumulator (tempoC+=tempoI; while>=150 tick; PPQN 24), gClockTable[49] for waits/note-lengths, running-status (bytes<0x80 reuse the last cmd>=0xBD), notes 0xCF-0xFF (ply_note operands key/vel/gate), FINE/GOTO/PATT/PEND/REPT, VOICE/VOL/PAN/BEND/BENDR/TUNE/TEMPO/KEYSH. Voicegroup ToneData(12B): DirectSound (type&7==0; type&0x08 = FIX = play at native rate, drums), keysplit(0x40)/drumkit(0x80) one level deep, CGB square/noise synthesised. WaveData: freq=1024*sampleRate, SIGNED 8-bit PCM, loop. pitch = freq/1024 * 2^((note+60-toneKey+keyShift-60)/12 + bend/tune). Mix stereo @13379Hz with ADSR (atk additive, dec/rel multiplicative ×/256, 60Hz frame) + linear interp; WAV→ffmpeg→opus. CCWriter emits per-map `backgroundsound="music/song-<id>.opus"`; main rips the maps' unique BGMs (most-used first, cap 64) — runs in DEFAULT and --all (music is map output). `--ripsong <id>` debug-rips one song.
* **Item icons:** gItemIconTable = {u32 image->LZ77, u32 palette} pairs; take the LONGEST run (a 49-entry decoy precedes the real ~376). Palette LZ77 OR raw 32B. FRLG+Emerald have pointer palettes (icons work); RSE Ruby/Sapphire store a palette TAG not a pointer (palette lives in a separate gItemIconSpritePalettes) -> no icons there yet (non-fatal client warnings). Emerald is a ruby SUB -> shares ruby's maps -> its music is ruby's (none ripped on the emerald run).
