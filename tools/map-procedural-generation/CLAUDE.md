# map-procedural-generation — rules

Combine with the root CLAUDE.md. This is the **content / assembly** stage: it consumes the base terrain from `../map-procedural-generation-terrain/` (THE standalone base terrain generator) and CHUNKS its big `all.tmx` into per-map **zlib**-base64 `.tmx`, then adds cities/roads, reciprocal borders + doors, NPCs/trainers/signs, wild-encounter tables and interiors (heal / shop / gym). Full on-disk spec = `target.md`; per-datapack tile/item/skin config = `settings*.xml` + `DATAPACK-CONFIG.md`.

* Output is a STAGING tree at `<binary>/dest/map/main/official/` — NEVER the source datapack (`CatchChallenger-datapack/` is read-only; the owner copies `dest/` over). `cities.txt`/`dialog.txt` load from CWD; settings come from `settings.xml` (or `--config settings-pkmn.xml`).
* **RIGID rectangular grid-aligned features** (root + terrain CLAUDE.md): the terrain generator shapes the grass/feature zones; FILL them solid, NEVER thin a zone with per-cell noise to look "natural" (the `doc/mapping/Badmap.png` mistake — reverted once already).
* **Build/run:** `cmake -S . -B <build> && cmake --build <build>` (out-of-tree). It ABORTS unless `<binary>/dest/map/tileset/` holds the `.tsx`+`.png` the chosen `settings.xml` names (it reads each tileset's tile count for `firstgid`). Stage by copying the datapack's `map/tileset/*` there. ~0.5 s, ≈55 tmx / 64 xml.
* **VERIFY by rendering** (vision): maps are **zlib** base64, NOT zstd — a zstd renderer shows all-black. Use a zlib tileset renderer that loads `.tsx`/`.png` from `dest/map/tileset/`. A city must read as a town (HEAL/SHOP/gym buildings, grid trees); roads carry rigid grass blocks.
* **Content quality:** NPC/trainer `<name>` come from an invented neutral name pool in `botStepXml` (LoadMapRoad.cpp), NOT the numeric bot id; flavor NPC lines come from `dialog.txt` (original themed sentences, not Lorem-Ipsum). `target.md` §2.1 = the region / location / `building/` naming taxonomy (real name → type fallback `city-N`/`cave-N`/`road-N`; isolated interiors under one `building/`; multi-region split by name/graph/`region-N`).
* **City/gym/cave settings:** `[city]` — avenue path-terrain is DERIVED from `template/city-big.tmx`/`city-medium.tmx` (Walkable fill = path, OnGrass 3x3 ring = border; edit the tmx to retheme; `useAsBase` brushes the tmx itself centered), per-type `signTiles` list (one sign style per city, signs at the avenue entrances, re-asserted post-vegetation so nothing hides them). Big city: STREET-FRONT LOTS filled center-out on both avenue frontages (doored buildings only on the door-facing side, door-front tiles validated+reserved), most houses doorless facades, plaza patches. `[building] gymTypes="type[:#color]->Mon1,..;.."` — typed gyms: part of the trainer teams from the type pool, and `tileset/gym.png` (lower parts = recolor mask at +128px) recolored per type-color into `dest/map/tileset/gym-<type>.*` (`typeXml` can point at the datapack `monsters/type.xml` for colors). `[road] cave\*` — that percent of road chunks: the OVERWORLD stays natural terrain (no road) with only mouth pockets (`entranceTile`) at the road connections; the corridor is the separate `<chunk>-cave[-N].tmx` `type="cave"` interior (wallTile = 9-tile 3x3 block, up to `maxDepth` mirrored floors linked by `stairDown/UpTile`, +2 monster levels & luck rotation per floor, `itemPercent`/`items`/`itemTile` ground items, trainers inside). `[wildMonsters] <id>\name` — generated xml references monsters by lowercase NAME; `minLevel==maxLevel` collapses to `level=`. `start.xml` gets one `<start>` per city in the lowest 10% of the level range (lowest = `Normal`; others = city-name ids, active once matching profiles exist in `player/start.xml`).
