# map-procedural-generation-terrain — rules

Combine with the root CLAUDE.md. This is **THE standalone base terrain generator** for the whole world — run it first, it must keep working on its own. Voronoi biomes + height/moisture noise + grid-aligned vegetation → ONE big base map `dest/map/main/official/all.tmx` (e.g. 176×176, **CSV**-encoded; layers `Water`/`Walkable`/`Grass`/`OnGrass*`/`LedgesDown|Left|Right`). The sibling `../map-procedural-generation/` then CHUNKS `all.tmx` into per-map zlib `.tmx` and adds cities/roads/bots/encounters on top — it does NOT regenerate terrain.

* **RIGID, RECTANGULAR, grid-aligned feature BLOCKS — never scattered/organic.** Grass, trees, water, ledges are solid grid-aligned rectangles (`doc/mapping/Goodmap.png` good vs `Badmap.png` bad): it IS the Pokémon GBA style AND compresses far better (long runs of one gid). **The grass here is already correct — leave it rigid.** NEVER thin a feature zone with per-cell noise to look "natural" (that is the Badmap mistake), here or downstream.
* **Stays standalone.** Builds out-of-tree (`cmake -S . -B <build> && cmake --build <build>`); must run to completion on its own. Run staging the binary expects under `<binary>/`: `dest/map/tileset/` holding the `settings.xml` tilesets (`animation.tsx` — note the datapack ships `animationS.tsx`, copy/rename — plus `terra.tsx`, `wood.tsx`, `invisible.tsx`); the `template/` dir (`tree-*.tmx`, `grass.tmx`, `flowers.tmx`); and a PRE-CREATED `dest/map/main/official/` dir (else the written maps' `../../tileset/...` refs can't resolve). Never write tilesets back into a source datapack tree (read-only).
